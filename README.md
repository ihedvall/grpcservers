# Google RPC Servers

## Summary

This repository implements Google RPS servers that are based on workflow tasks. 
The reason is that the gRPC libraries are challenging to link with libraries that use the OpenSSL encryption. 
Google is using another encryption library that doesn't link with the OpenSSL. 
This is possible to solve the problem, but the linking of special build gRPC libraries is problematic.

The repository is a library that implements the gRPC C++ server but also implements its client C++ interface.

The repository implements the following servers.
- **Event Log gRPC Server**.

## Event Log gRPC Server
The event log gRPC server is a server that implements a syslog data server.
The server and its database are designed to store events generated by external applications.
It isn’t intended to be used for general syslog events.

The server uses an SQLite database for storing events.
The server creates the database at the startup if it is missing.
Note that the event log service exists without a gRPC interface 
in repository [Event Log](https://github.com/ihedvall/eventlog).

## Building the Library and Servers
Building can be challenging as the gRPC server doesn’t use OpenSSL encryption.
The build won’t work with the VCPKG package manager.
Instead, it is the component directory (COMP_DIR) build used?

The services are pre-built for 64-bit Windows.
